\begin{abstract}

We are interested in researching a game theoretical scenario for which no obvious equilibrium state exists. Specifically, we decided to look at a series of Colonel Blotto games, both the canonical game as well as a game with a retaining structure. We wanted to see what controllers could primarily maximize utility, and secondarily minimize the game-to-game variance.

\end{abstract}

\section{INTRODUCTION}

Consider a classic variation of colonel blotto game with two battlefields and two generals, one defending general and one attacking general. Both generals have n troops and must decide how to allocate these troops between the two battlefields. For each battlefield, the defender wins if they have at least as many troops as the attacker (meaning they win in a tie) and the attacker wins if they have more troops than the defender. The total utility of the game is 1 if the defender defends both battlefields, 0 if they defend one of the battlefields, and -1 if they defend neither battlefield. It is worth noting that a utility of -1 isn’t possible in the classic game, as when attacker and defender have the same troop count, it’s impossible for the attack to win both fields. The defending general wants to maximize utility while the attacking general wants to minimize it, and they must simultaneously commit to a strategy without knowing the strategy of the other general.
The defending troops will do whatever they are told. The attacking troops, however, receive utility m for obeying orders (as opposed to defecting) and are penalized utility p if they are captured during an attack. If X is the perceived mixed strategy of the attacking general and Y is the perceived mixed strategy of the defending general, the attacking troops will defect if the expected capture probability C(X, Y) <= $\vartheta$, where $\vartheta$ is equal to m/p. The defending general may manipulate their mixture Y in order to take advantage of this deterrence dynamic. The attacking general must be careful about their own mixed strategy in order to prevent their troops from defecting.
When $\frac{1}{2} \leq \vartheta < \frac{3\lfloor n/2 \rfloor+1}{4\lfloor n/2 \rfloor+2}$ (which goes to $\frac{3}{4}$ in the limit), there do not exist Nash equilibrium values of X and Y. In other words, it is not clear what rational generals would do in this situation.

\subsubsection{Random}
The Random agent selects uniformly at random from all valid troop allocations available to it. The agent does not attempt to model the opponent or reason about expected outcomes. Instead, it acts as a baseline policy for comparison with more sophisticated agents. Because its behavior is entirely stochastic, its performance provides a lower-bound benchmark for evaluating the effectiveness of the other agents.
\subsubsection{Even}
The Even agent deterministically divides its troops evenly across the two battlefields. If the total number of troops is not divisible by two, remaining troops are sent to the right battlefield. This agent represents a simple heuristic strategy that attempts to maintain balanced coverage of all battlefields without adapting to the opponent's behavior. It serves as a deterministic baseline against which more adaptive strategies can be compared.
\subsubsection{Monte Carlo}
The Monte Carlo agent is designed for the non-retaining variant of the simulation. In this setting, the perceived capture probability is ignored to simplify the evaluation process. This simplification reduces the complexity of the state space and allows the agent to evaluate all possible action combinations for the next turn.

Rather than simulating entire games, the agent performs a one-step lookahead search. For each possible move available to the agent, the algorithm enumerates all possible combinations of moves by both players in the next turn. The resulting utility of each combination is computed, and the utilities corresponding to each candidate move are aggregated. The agent then selects the move that maximizes the total utility across all combinations that include that move. If multiple moves produce the same utility, one is selected uniformly at random.

Although this agent is referred to as a Monte Carlo agent, it does not perform stochastic sampling of actions. Traditional Monte Carlo methods estimate expected outcomes through repeated random sampling of possible scenarios. In contrast, this agent deterministically enumerates all possible one-step action combinations and therefore does not constitute a true Monte Carlo simulation. Instead, it serves as a simple baseline evaluation method for the non-retaining variant of the game.
\subsubsection{Monte Carlo Tree Search}
This agent can be viewed as an extension of the Monte Carlo agent designed to handle the additional complexity introduced by troop retention. Because this additional complexity makes flat Monte Carlo evaluation impractical, we instead employ a Monte Carlo Tree Search (MCTS) algorithm to explore possible actions and estimate their expected utility.

The search tree represents sequences of moves for the agent acting as either the attacker or defender. At each node corresponding to our agent's decision, possible troop allocations are expanded as child nodes. The opposing player's action is modeled using a stochastic policy, where the opponent's move is selected randomly. Future work could improve this component by incorporating a more accurate model of opponent behavior.

As shown in figure~\ref{fig:mcts_tree}, Monte Carlo Tree Search consists of four phases: selection, expansion, simulation, and backpropagation.

\paragraph{Selection and Expansion}
The search proceeds for a fixed computational budget defined by a maximum number of node expansions (typically around 100 in our experiments). During each iteration, the algorithm traverses the existing tree from the root using a modified Upper Confidence Bound (UCB) selection rule. The UCB value for each node is computed from the average observed utility and the number of times the node has been visited relative to its parent, balancing exploration of under-sampled nodes with exploitation of high-performing ones.

\paragraph{Simulation and Backpropagation}
Once a leaf node is selected, the algorithm performs several random rollout simulations from that state to estimate the expected utility of the position. The resulting utility is then backpropagated through the visited nodes, updating their statistics.

After the search budget is exhausted, the agent selects the action corresponding to the child of the root node with the highest visit count, which represents the branch with the strongest empirical performance.

\subsubsection{Retaining Heuristic}
The Heuristic agent prioritizes securing a battlefield outright before distributing troops more evenly. Concretely, when the agent does not yet have enough remaining troops to guarantee victory on both battlefields, it allocates all available troops to one battlefield chosen uniformly at random. Once its remaining troop count is sufficiently large that it could allocate equal numbers of troops to both battlefields and still exceed the opponent’s previous allocations, the agent switches to an even split across both battlefields.

\subsection{Monte Carlo Based Controllers}
An interesting behavior was observed with the Monte Carlo (MC) controller. In scenarios where the number of attackers and defenders was equal, the MC controller behaved identically to the Random controller. This occurs because, under these conditions, the Monte Carlo sampling procedure produced uniform utility estimates across the available actions. As a result, the policy induced by the MC controller was effectively equivalent to selecting actions uniformly at random.

This observation suggests that, in non-retaining games of this type, uniform random action selection can constitute a viable baseline strategy when the expected utilities of the available actions are symmetric. In other words, when no action yields a consistently higher payoff under the sampled rollouts, the MC controller does not develop a meaningful preference and defaults to behavior indistinguishable from random play.

A second observation arises in cases where the number of attackers exceeds the number of defenders. Under these conditions, the Dynamic Programming (DP) Exploit controller was able to achieve higher performance against the MC controller than against the Random controller. This result is somewhat counterintuitive, as the Exploit controller was explicitly designed to exploit uniformly random policies. One possible explanation is that the MC controller introduces subtle biases in action selection due to sampling variance or rollout structure, which the exploitative strategy can leverage more effectively than pure uniform randomness.

The Monte Carlo Tree Search (MCTS) controller exhibited highly variable performance across trials. In some cases, it achieved strong outcomes, while in others it performed significantly worse than the simpler baseline controllers. One possible explanation for this inconsistency is the simulation policy used during the expansion and rollout phases of the MCTS algorithm. If the rollout simulations do not accurately reflect the true value of game states, the resulting value estimates may misguide the tree search process. Alternatively, the observed variability may be attributable to implementation errors or insufficient exploration within the search tree. Further investigation is required to determine the primary cause of this behavior.
